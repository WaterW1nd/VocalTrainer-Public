# Руководство по подготовке трека — TOOLS_GUIDE.md

Серверная часть запускается в **Docker-контейнере**. Нужно установить только Docker Desktop и скачать файлы моделей.

---

## Шаг 1 — Установить Docker Desktop

1. Перейди на **https://www.docker.com/products/docker-desktop**
2. Нажми **Download for Windows**, установи, **перезагрузи ПК**
3. Запусти Docker Desktop — в трее должна появиться иконка кита 🐳

---

## Шаг 3 — Сборка и запуск

Открой **PowerShell**, перейди в папку `server\` и выполни:

```powershell
cd C:\karaoke-trainer\server
docker compose up -d --build
```

Первая сборка: ~5–10 минут (устанавливает Python-пакеты и копирует модели).
Последующие запуски: мгновенно.

---

## Шаг 4 — Открыть приложение

1. Открой `index.html` в Chrome или Edge (**из любого места** — серверная часть работает независимо)
2. На главном экране нажми **«Подготовка трека»**
3. Зелёный кружок **● Сервер работает** — всё готово

## Управление контейнером

```powershell
# Из папки server\:
docker compose up -d            # Запустить в фоне
docker compose down             # Остановить
docker compose logs -f          # Смотреть логи в реальном времени
docker compose restart          # Перезапустить (после замены tools_server.py)
docker compose up -d --build    # Пересобрать образ (после изменений Dockerfile)
```

## Ползунки и параметры

### 🎼 Создание MIDI

| Ползунок | Диапазон | По умолчанию | Что делает |
|---|---|---|---|
| Чувствительность нот | 0.3 – 0.9 | **0.80** | Порог обнаружения ноты. Выше → меньше ложных нот в тишине |
| Минимальная длина | 50 – 600 мс | **380 мс** | Фильтр коротких нот. Выше → убирает артефакты |

> **Совет:** Если MIDI содержит много «мусора» в тишине — увеличь чувствительность до 0.85–0.9 и длину до 400–500 мс.

### ⚙️ Настройка MIDI (пост-обработка)

| Ползунок | Диапазон | По умолчанию | Что делает |
|---|---|---|---|
| Объединять паузы | 0 – 1.0 с | **0.40 с** | Сшивает ноты одной высоты с паузой ≤ значения |
| Удалять ноты короче | 0 – 0.5 с | **0.30 с** | Убирает очень короткие ноты-артефакты |

Блок **«До / После»** показывает количество нот до и после применения фильтров.

## Настройка производительности

В `docker-compose.yml` можно настроить:

```yaml
environment:
  DEMUCS_THREADS: "2"        # Потоки CPU для Demucs (меньше = меньше нагрузка)
  DEMUCS_TIMEOUT_SEC: "2700" # Таймаут 45 мин (kill при зависании)
  MAX_JOB_AGE_SEC: "7200"    # Файлы удаляются через 2 ч
```

Если хочешь снизить нагрузку на CPU ещё больше:
```yaml
DEMUCS_THREADS: "1"  # ~50% CPU вместо ~85%, но медленнее в 1.5-2х
```

## Проблемы и решения

| Проблема | Решение |
|---|---|
| Кружок красный, сервер недоступен | Выполни `docker compose up -d` из папки `server\` |
| Красный ненадолго во время обработки | Нормально — сервер занят, ждёт освобождения потока. Подождать 20–30 с |
| Ошибка «invalid hash» при сборке | Модели повреждены — перескачай и положи заново в `models\` |
| Кнопка «Разделить» зависла на 85% надолго | Нормально для длинных треков. Если > 45 мин — `docker compose restart` |
| Контейнер падает с OOM | Docker Desktop → Settings → Resources → Memory → 6 GB+ |
| Порт 5050 занят | В `docker-compose.yml` измени `"5050:5050"` на `"5051:5050"` и обнови `TOOLS_SERVER_URL` в `index.html` |
| Кнопки «Разделить»/«Создать MIDI» неактивны | Убедись что файл выбран (зелёная рамка дроп-зоны) — кнопка активируется при следующем пинге (~10 с) |
| «Назад» во время обработки | Появится диалог подтверждения. При выходе обработка на сервере остановится (SIGTERM) |
